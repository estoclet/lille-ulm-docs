# Principes et perimetre

## Finalite

Le futur site doit permettre de :

- presenter clairement les offres ;
- vendre si besoin des prestations ou bons cadeaux ;
- gerer facilement les contenus et pages ;
- donner au gestionnaire un back-office rassurant, guide et simple ;
- structurer la croissance : leads, ventes, relances, clients, offres.

## Hypothese produit

Le site cible n'est pas un simple site vitrine. Il doit etre pense comme :

- un outil d'acquisition ;
- un support de vente ;
- un point d'entree CRM léger ;
- un espace d'exploitation quotidienne.

## Position Drupal

Le socle cible est :

- **Drupal 11** si possible ;
- **Drupal Commerce** si la vente en ligne reste dans le perimetre ;
- **Layout Builder + UI Suite + SDC** comme base de modularite des pages.

## Principe cle

Ne pas empiler plusieurs systemes de composition editorial sans raison.

Le systeme principal doit etre :

- **Layout Builder** pour assembler les pages ;
- **UI Suite / composants** pour garder une bibliotheque visuelle coherente ;
- des blocs et composants metier reemployables ;
- des formulaires et workflows simples pour le gestionnaire.

## Ce que ce cadre privilegie

- peu de concepts ;
- peu de clics ;
- une edition guidee ;
- des composants reutilisables ;
- des tableaux de bord orientes action ;
- des taches IA bien bornees.

## Ce que ce cadre evite

- de gros documents melant tout ;
- des speculations techniques sans decision metier ;
- des pages "full custom" non maintenables ;
- un back-office Drupal brut expose tel quel au gestionnaire ;
- deux ou trois facons differentes de modifier la meme chose.

## Question structurante

Le choix **Drupal Commerce** ne depend pas d'une mode technique mais d'une reponse claire a ces questions :

1. faut-il vendre en ligne ?
2. faut-il encaisser directement ?
3. faut-il suivre commandes, bons, remboursements, relances ?
4. faut-il segmenter les clients et les recontacter ?

Si la reponse est majoritairement **oui**, Drupal Commerce est pertinent.

## Cible editoriale

Le gestionnaire doit pouvoir :

- creer ou modifier une page sans casser le theme ;
- changer l'ordre des blocs ;
- remplacer une image ou une video ;
- mettre a jour des tarifs ou messages ;
- lire les ventes et les leads ;
- comprendre quoi faire sans jargon Drupal.
