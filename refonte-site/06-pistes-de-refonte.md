# Pistes de refonte

## Axe 1 : clarifier le besoin produit

Trois scenarios doivent etre arbitres rapidement :

### Scenario A : site vitrine seulement

- conservation des contenus utiles ;
- remplacement de la reservation en ligne par un formulaire ou une prise de contact ;
- archivage a froid des donnees `DT Register`.

### Scenario B : site vitrine + prise de reservation simple

- catalogue d'offres modernise ;
- demandes de reservation non transactionnelles ;
- back-office leger pour suivi manuel.

### Scenario C : site vitrine + vente / reservation structuree

- fiches offres ;
- tunnel d'achat ou de reservation ;
- paiements ;
- confirmations ;
- calendrier / disponibilites ;
- reprise partielle d'historique.

Le site actuel laisse penser que le vrai besoin historique se situe entre B et C.

## Axe 2 : reconstruire l'experience visiteur

Arborescence cible minimale recommandee :

- Accueil
- Offres / Tarifs
- L'experience
- Galerie
- FAQ
- Nous trouver
- Contact
- Mentions legales

Contenus a valoriser :

- baptemes de l'air ;
- initiation au pilotage ;
- localisation Bondues / Lille ;
- preuves visuelles ;
- reassurance paiement / securite ;
- telephone de contact.

## Axe 3 : rationaliser la stack

Principes recommandes :

- ne pas reconduire Gantry 4 / RocketTheme ;
- ne pas reconduire BreezingForms ni `DT Register` sans validation explicite ;
- repartir sur une base moderne, legere et documentee ;
- isoler la reprise de donnees comme un chantier specifique.

## Axe 4 : traiter la donnee comme un lot separe

Lots a prevoir :

- donnees clients / inscriptions historiques ;
- catalogue d'offres ;
- contenus editoriaux ;
- medias images / video ;
- SEO de base : URLs, meta, redirections.

## Priorites de travail recommandees

1. Valider le scenario produit cible pour la reservation.
2. Figer l'arborescence et les contenus a reprendre.
3. Extraire et qualifier les medias utiles.
4. Definir la politique de reprise / archivage des donnees historiques.
5. Concevoir la nouvelle UI et les parcours prioritaires.
